# Renderer: Zensical (Material for MkDocs team's successor SSG).
#
# Why Zensical, not mkdocs-material?
#   * mkdocs-material 9.7.x is on maintenance-only support until 2026-11.
#   * The PyPI ownership change on 2026-03-09 broke the original mkdocs
#     governance; the rewrite (MkDocs 2.0) drops plugins + theming with
#     no migration path. The Material team responded by leaving the
#     mkdocs ecosystem and starting Zensical from scratch.
#   * Zensical ships a Material-like theme with the same config knobs
#     (site_name, theme.features, theme.palette, markdown_extensions.*)
#     so the migration was essentially a config-format port.
#
# Caveats:
#   * Zensical is alpha (0.0.x). Pin tightly; review every minor bump.
#   * Some Material-era theme features (icon names, colour aliases) may
#     render differently in alpha until Zensical hits a stable release.
#   * musllinux_1_2 wheels exist for amd64 + aarch64, so install is
#     fast on Alpine without a Rust toolchain.
zensical==0.0.43
pymdown-extensions==10.21.3
# tomli-w lets us rewrite the user's zensical.toml at runtime so docs_dir
# and site_dir always point at the container paths the add-on relies on,
# regardless of what the user wrote for their GitHub Pages workflow. Pure
# Python wheel, ~30 KB, no compilation needed.
tomli-w==1.2.0
